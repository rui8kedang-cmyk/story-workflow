## 项目启动：艾迪小发明家 - AI动画短片系列

### 角色设定
- **艾迪**：小学生，小发明家。聪明、邋遢、鬼点子多、容易弄巧成拙。学习成绩普通，喜欢用发明解决问题。
- **爷爷**：知名发明家。聪明、古怪孤僻、脾气倔、老顽童、有点好色。

### 核心设定
- 艾迪的发明零件都来自爷爷丢掉的垃圾
- 故事围绕日常生活/学习中的难题 → 发明创造 → 有时成功但大多闹笑话
- 风格：迪士尼/皮克斯 3D动画风格
- 视频长度：45秒 - 1分30秒
- 目标人群：全年龄

### 工作流程
1. 每轮生成10个故事
2. 用户选择有趣、童真、正能量、可用的故事
3. 分析选择偏好，迭代优化
4. 积累5个可用故事后，生成分镜剧本（适配即梦 Seedance 2.0 提示词）

### 进度

- 第2轮：故事④「作业速写笔」✅采用 | 故事⑦「自动浇花器」主题对但结局不够有趣 | 其余淘汰

### 第2轮分析
采用故事特征（作业速写笔）：
- 难题非常具体、贴近小学生日常（抄课文三遍）
- 发明有巧妙的机械逻辑（一支笔带动三支笔同步书写）
- 过程有"先成功"的铺垫（前几行完美）
- 结局笑点是视觉化的渐变过程（偏移越来越大→写到同桌桌上）
- 结局不是"爆炸式"灾难，而是"慢慢跑偏"的荒谬
- 无伤大雅，没有人受伤或生气

故事⑦问题：主题好（忘浇花→自动浇灌），但结局太散——浇了衣服、浇了邻居、浇了自己，笑点分散，没有一个集中的"打脸"画面

淘汰故事共同问题：
- 有些结局太暴力/夸张（弹飞出去、满脸红烧肉）
- 有些结局靠"量变"不靠"质变"（吸了一堆金属、泡泡溢出来）
- 后半段出现错别字和表述错误——必须严格杜绝

第3轮优化方向：
1. 难题选择更日常、更贴近小学生真实痛点
2. 发明的机械逻辑要巧妙有趣
3. 结局追求"慢慢跑偏"的渐变式荒谬，而不是"一下子爆炸"
4. 一个集中的视觉笑点，不要分散
5. 严格检查错别字和表述
问题：
1. 不能出现空白/无内容的故事
2. 故事结构不对——需要严格的四段式：难题→发明→过程→结局
3. 发明描述要具体：明确写出用了什么零件、怎么组装的，让人能"看到"这个装置
4. 过程要有画面感：描述一个关键的动态场景
5. 结局要有"打脸"的喜剧效果：发明反噬到艾迪自己身上，形成视觉笑点
6. 不要加爷爷的点评/金句——爷爷不是每个故事都要出场
7. 不要用文字梗（反字、密码之类），要用视觉笑点（拍脸印、飞出窗户等）
8. 语气不要刻意搞笑，让情节本身好笑

### 第3轮新增教训
- 家里不要出现其他亲属角色，艾迪和爷爷两人生活
- 结局要有足够反转和笑点，不能只是"没了/坏了"这种平淡收尾
- "越来越懒"的设计思路可以用，但结尾必须有视觉笑点
- 发明本身要有亮点/巧思，不能太普通（如普通拖把加马达）
- 好的结局模式：书包完美vs自己狼狈（对比反差）、机器停不下来→人被迫配合（人机错位）

### 范例故事分析：《蚊子终结者》
结构：
- **难题**：一句话，具体生活场景（蚊子嗡嗡叫睡不着）
- **发明**：具体零件+组装方式（机械平台+激光瞄准器+苍蝇拍+机械手）
- **过程**：一个关键画面（激光点在黑暗房间里疯狂移动）
- **结局**：发明精准运行但结果荒谬（打中了艾迪自己的脸，蚊子飞走）

关键特质：
- 发明逻辑上是对的（热感应+瞄准+自动打击）
- 笑点来自"机器太忠实地执行任务"导致的意外
- 结局是视觉化的（脸上的拍印、蚊子飞走）
- 没有多余角色、多余对话
- 短小精悍、一个场景、一个笑点
