## 眼睛动画集成到小智固件 ✅
### 最终方案：LVGL Screen 切换
经历多轮迭代：overlay方案 → screen切换方案

**overlay方案踩坑记录：**
1. ShowNotification("固件版本2.0.5") 会停止眼睛 → 修复：notification不触发stop
2. 一屏有眼睛另一屏黑屏 → lv_display_get_next顺序不对 → 改用GetLvDisplay()直接获取
3. 眼睛不动 → lv_obj_align不触发重绘 → 改为lv_obj_set_pos + lv_obj_invalidate
4. overlay被小智UI刷新覆盖 → 根本问题：overlay方案在多display场景不可靠

**最终方案：为每个display创建独立LVGL screen**
- `lv_obj_create(NULL)` 创建新screen（不是overlay子对象）
- `lv_screen_load()` 切换屏幕（自动触发全屏刷新）
- 待机：切到eye screen；唤醒：切回xiaozhi orig screen
- ✅ 双屏都正确显示眼睛，动画运行正常

### 实现文件
- `animated_eyes.h` / `animated_eyes.c` — LVGL v9 screen-based 眼睛动画
- `display_manager.h` / `.cc` — InitAnimatedEyes + SetEmotion控制
- `esp32-s3-dualeye-lcd-1.28.cc` — 构造函数调用InitAnimatedEyes

### 关键教训
- LVGL v9多display场景用screen切换，不要用overlay
- `lv_screen_load(scr)` 加载到scr所属的display
- `lv_obj_create(NULL)` 在当前default display上创建screen
- timer callback在LVGL task内运行，不需要lvgl_port_lock

### 验证结果
- ✅ 双屏黑底 + 白色巩膜 + 棕色虹膜 + 黑色瞳孔 + 高光
- ✅ 眼珠随机转动 + 周期性眨眼
- ✅ 对话过程中眼睛隐藏，显示小智对话 UI
- ✅ 对话结束回待命，双屏眼睛自动恢复
- ✅ 多轮对话不卡死，系统稳定

### 踩坑总结（v1→v2→v3）
**v1 Overlay方案（失败）：** 用lv_obj在active screen上叠加，被小智UI覆盖
**v2 Screen切换方案（部分成功）：** 用lv_screen_load切换独立screen
  - 双屏显示OK但动画不动（lv_obj_set_pos不触发非默认display刷新）
  - 改canvas重绘后双屏有眼睛有动画
  - 但对话结束切回时一个屏卡住（lv_screen_load对第二个display不可靠）
  - stop中的vTaskDelay阻塞了小智状态机导致卡死
**v3 Overlay+永驻Task（成功）：** canvas overlay在小智screen上，用HIDDEN控制
  - anim_task永远运行，flag控制show/hide
  - start/stop只设布尔值，零阻塞
  - 不做screen切换，不改default display

### 关键教训
- LVGL v9多display的lv_screen_load不可靠用于频繁切换
- canvas每帧重绘（lv_canvas_fill_bg + lv_draw_rect）是最可靠的动画方式
- 在ESP嵌入式环境中，display操作绝不能阻塞调用者
- 用flag+永驻task模式比start/stop task模式更稳定

## 舵机控制：PCA9685 + SG90

### 接线方案
DualEye 板有两个 SH1.0 14PIN 接口，都是空闲扩展口（屏幕通过内部排线连接）

LCD1-Board 接口引脚定义：
- Pin 1: USB_5V | Pin 2: GND | Pin 3: D_N | Pin 4: D_P
- Pin 5: 3V3 | Pin 6: GND | Pin 7: SDA | Pin 8: SCL
- Pin 9: UART_RXD | Pin 10: UART_TXD
- Pin 11: TP2_SDA | Pin 12: TP2_SCL | Pin 13: GPIO0 | Pin 14: RESET

PCA9685 接线：Pin1→V+, Pin2→GND, Pin5→VCC, Pin7→SDA, Pin8→SCL

### 进展
- ✅ PCA9685 I2C 通信成功（地址 0x40，Prescale=121, 50Hz）
- ✅ 与 ES8311 音频芯片共享 I2C 总线无冲突
- ❌ 舵机未动：疑似 V+ 5V 供电未接通（排线 Pin 编号方向待确认）
- 需要公对公杜邦线或万用表确认排线引脚
- ⚠️ **关键发现（4/14 商家技术支持确认）：ESP32-S3 工作电压 3.3V，GPIO 最高输出也是 3.3V**
  - 排线 Pin1 标注 USB_5V 但实际可能只在 USB 供电时才有 5V
  - PCA9685 VCC 可接 3.3V（逻辑电压），但舵机 V+ 必须外部供 5V
  - SG90 舵机需要 4.8-6V 驱动，3.3V 肯定不够

⚠️ 注意：LCD2-Board 接口包含屏幕控制信号，拔掉会导致黑屏！
